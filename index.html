1) Benefits of scalable computing over the Internet:

  1. Elasticity and On-demand Resources:
    • Cloud systems automatically add or remove resources based on the user demands.
    • You get more resources when demand is high,
      you get necessary resources when demand is low.
  2. Cost Effectiveness:
    • Instead of buying external computer hardware or building data centers to store data,
      users can simply pay rent for storing the data in cloud servers.
    • If you use more, you pay more,
      if you use less, you pay less.
  3. Scalability:
    • The ability of a computing system to handle workloads by adjusting its performance and costs is called scalability.
    • Instead of upgrading the entire system, cloud systems allow you to add more servers to increase scalability.
  4. Availability & Reliability:
    • Cloud systems store data at different locations around the world.
    • If one data center fails, the data is still safe in the cloud system, which makes it more reliable and highly available.
  5. Faster Deployment & Agility:
    • Cloud systems allow you to load applications to cloud servers quickly.
    • We can quickly create virtual machines and deploy the software without the need for any hardware.

3) Explain how scalable computing over the internet improves fault tolerance and reliability in cloud computing

  • Scalable computing in cloud systems distributes workloads across many computers and virtual machines.
  • It eliminates single points of failure; i.e., if one computer or virtual machine fails, others can automatically take over.
    1. Redundancy:
      • Cloud system uses multiple computers and virtual machines to store the data and applications.
      • If one computer or virtual machine fails, others can automatically take over.
      • This makes data and applications always available.
    2. Real Time Resource Monitoring:
      • Cloud system allows monitoring the resources in real-time.
      • We can reassign the workloads as needed.
    3. System Availability:
      • A system is highly available if it has a long MTTF and short MTTR.
      • MTTF stands for Mean Time to Failure
        MTTR stands for Mean Time to Repair

        • System Availability = MTTF MTTF + MTTR

    4. Horizontal Scaling:
      • Adding more nodes to virtual machines boosts performance but also spreads the risks.
      • So failure in one node does not affect the entire system.

8)Compare and contrast between centralized, decentralized and distributed system models:

Centralized System Models:

  • In centralized system models, all resources are controlled by a single node
  • High chances of failure, if the one point fails then all the connected nodes will be failed
  • Centralized system models show poor scalability
  • Centralized system models are simple to manage and control
  • Centralized system models show low performance

Decentralized System Models:

  • In decentralized system models, all resources are controlled by multiple nodes
  • Here, not all nodes are interconnected
  • Moderate chances of failure
  • Decentralized system models show better scalability than centralized system models
  • Decentralized system models are complex to manage and control
  • Decentralized system models show moderate performance

Distributed System Models:

  • In distributed system models, all resources are controlled by multiple interconnected nodes
  • Low chances of failure, if a single node fails then the resources are still safe in other nodes
  • Distributed system models are highly scalable
  • Distributed system models are very complex to manage and control
  • Distributed system models show high performance

11)MapReduce Model:

  • MapReduce Model is a programming model developed by Google.
  • MapReduce Model helps to process the large amount of data across many distributed systems.
  • There are mainly three phases in MapReduce Model:
    1. Map phase
    2. Shuffle and Sort phase
    3. Reduce phase

    Map Phase:

    • The input dataset is divided into smaller parts.
    • Each part is processed independently and runs at the same time.
    • It converts the data into key-value pairs.

    Shuffle and Sort Phase:

    • The system collects all key-value pairs from the Map Phase.
    • It groups the values by their keys.
    • Keys are sorted for organizing the data.

    Reduce Phase:

    • Each Reducer gets a key and its associated values.
    • It performs aggregate operations like count(), sum(), avg()...
    • After performing this operation, the result is generated as Key-Value Pairs.

  4. DoS Attack:

1) Benefits of scalable computing over the Internet:

  1. Elasticity and On-demand Resources:
    • Cloud systems automatically adds or remove resources based on the user demands
    • You get more resources when demand is high,
      You get necessary resources when demand is low
  2. Cost Effectiveness:
    • Instead of buying External Computer Hardwares or Building data centers to store the data
      Users can simply pay the rent for stroing the data in cloud servers
    • If you use more,You pay more,
      If you use less,You pay less
  3. Scalability:
    • The ability of a computing system to handle the workloads by adjusting it's performance and costs is called Scalability
    • Instead of Upgrading the entire system,Cloud System allows you to add more servers to increase scalability
  4. Availability & Reliability:
    • Cloud System stores the data at different locations around the world
    • If one data center fails,the data is still stafe in cloud system which makes it more reliable and highly available
  5. Faster Deployment & Agility:
    • Cloud System allows you to load the Applications to cloud servers fastly
    • We can quickly create Virtual Machines and Deploy the Software without need of any Hardware.

3)Explain how scalable computing over the internet improves fault tolerance and reliability in cloud computing

  • Scalable Computing in Cloud Systems distributes work-loads across many computers and virtual machines.
  • It eliminates single points of failure; i.e., if one computer or virtual machine fails, others can automatically take over.
    1. Redundancy:
      • cloud system uses multiple computers and virtual machines to store the data and applications.
      • if one computer or virtual machine fails, others can automatically take over
      • This makes data and Applications are always available.
    2. Real Time Resource Monitoring :
      • Cloud System allows to monitor the resources in Real-time.
      • We can reassign the workloads as needed.
    3. System Availability :
      • A System is highly available if it has a long MTTF and short MTTR.
      • MTTF stands for Mean Time to Failure
        MTTR stands for Mean Time to Repair

        • System Availability = MTTF MTTF + MTTR

    4. Horizontal Scaling :
      • Adding more nodes to a virtual machines boosts the performance but also spread the risks.
      • so failure in one node do not affect the entire system.
